很开心可以继续分享这个话题，下一个要讨论的就是事件驱动，将使我们实现的婴儿虚拟dom运用到真实的项目开发。

今天我们主要讨论的的是setting／diffing属性（props）和事件处理。

babel处理

在这之前我么处理一个普通的标签

<div></div>

通过babel处理会转化成props为null，因为没有属性，结果如下：

&#123; type: 'div', props: null, children: [] &#125;

我们最好将它（props）处理成一个空的对象，因为我们将要对props进行迭代取值

function h(type, props, ...children) &#123;
    return &#123; type, props: props || &#123;&#125;, children &#125;;
&#125;

设置简单的属性，比如下面的，会在我们的dom中怎么展现呢？我们将它存在一个对象里。

<ul className="list" style="list-style: none"></ul>

我们在内存中将在如下展示

&#123;
    type: 'ul',
    props: &#123; className: 'list', style: 'list-style: none' &#125;,
    children: []
&#125;

我们取对象的属性值给我们真实的节点设置属性。

fucntion setProp($target, name, value) &#123;
    $target.setAttribute(name, value);
&#125;

我们知道了设置一个属性，设置所有属性只不过是遍历一下props对象。

function setprops($target, props) &#123;
    Object.keys(props).forEach((name) => &#123;
        setProp($target, name, props[name])
    &#125;)
&#125;

现在还记得createElement()函数吗？我们仅仅在创建珍视的dom的时候给节点设置属性。

function createElement(node) &#123;
    if (typeof node === 'string') &#123;
        return document.createTextNode(node)
    &#125;
    const $el = document.createElement(node.type)
    setProps($el, node.props)
    node.children
        .map(createElement)
        .forEach($el.appendChild.bind($el));
    return $el
&#125;

但是这没有结束，我们遗漏了一些细节，首先class在javascript是个关键字，所有我们不能用class，我们用className代替。

<nav className="navbar light">
  <ul></ul>
</nav>

但是我们在真实的dom里面没有className属性这个值，所有我们要通过setProps（）函数处理。还有就是属性值设置成布尔值代表属性存在与否。

<input type="checkbox" checked=&#123;true&#125; />
